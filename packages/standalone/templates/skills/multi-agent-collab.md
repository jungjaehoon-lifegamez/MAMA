# 멀티에이전트 협업 스킬

Skill Forge의 오케스트레이터 패턴을 활용하여 에이전트간 역할을 동적으로 배정하고 협업 작업을 수행합니다.

## 역할 정의

### 🎯 Orchestrator (조율자)

- 작업을 단계별로 분해
- 에이전트에게 역할 배정
- 진행 상태 추적 및 핸드오프 관리

### 🔧 Developer (개발자)

- 실제 코드/스킬 작성
- Architect 설계 또는 Orchestrator 지시에 따라 구현
- 테스트 코드 포함

### 📝 Reviewer (리뷰어)

- 코드 품질 검증
- 보안/성능/가독성 체크
- 점수 기반 피드백 제공

## 워크플로우

```
┌──────────────────────────────────────────┐
│           Orchestrator                     │
│   1. 작업 분석 및 단계 분해                 │
│   2. 역할 배정 (Dev/Review)                │
│   3. 핸드오프 관리                          │
└─────────────┬────────────────────────────┘
              │
    ┌─────────┴─────────┐
    ▼                   ▼
┌─────────┐       ┌─────────┐
│Developer│ ────→ │Reviewer │
│(코드작성)│       │(코드리뷰)│
└─────────┘       └────┬────┘
    ▲                   │
    └───────────────────┘
        피드백 루프
```

## 사용법

### 기본 협업 요청

```
/collab [작업 설명]
```

### 역할 지정 협업

```
/collab --dev [에이전트A] --review [에이전트B] [작업 설명]
```

### 예시

```
/collab --dev DevBot --review Reviewer 헬스체크 스크립트 작성
```

## 협업 프로토콜

### Phase 1: 계획 (Orchestrator)

1. 작업 요구사항 분석
2. 필요 파일/모듈 식별
3. 단계별 작업 계획 수립
4. 에이전트 역할 배정

### Phase 2: 구현 (Developer)

1. Orchestrator 계획에 따라 코드 작성
2. 파일 시스템에 코드 저장
3. 기본 테스트 실행
4. 완료 시 Reviewer에게 핸드오프

### Phase 3: 리뷰 (Reviewer)

1. 파일 시스템에서 코드 직접 읽기
2. 체크리스트 기반 검증:
   - [ ] 코드 구조 및 가독성
   - [ ] 에러 핸들링
   - [ ] 타입 안전성
   - [ ] 보안 이슈
   - [ ] 성능 고려사항
3. 점수 (1-10) 및 피드백 제공
4. Critical 이슈 시 Developer에게 재작업 요청

### Phase 4: 반복 (필요시)

- Reviewer 피드백 → Developer 수정 → 재리뷰
- 10/10 또는 승인까지 반복

## 핸드오프 형식

### Developer → Reviewer

```
📝 리뷰 요청!
파일: [파일 경로]
변경 내용: [요약]
```

### Reviewer → Developer

```
🔴 Critical: [수정 필요 사항]
🟡 Suggestion: [개선 제안]
💡 Nitpick: [사소한 개선점]
📊 점수: X/10
```

## 파일 공유 규칙

- 같은 프로젝트 폴더를 공유하므로 **메시지에 코드 복붙 불필요**
- 파일 경로만 공유하면 상대 에이전트가 직접 읽기 가능
- 수정 사항은 파일에 직접 반영

## 주의사항

- Discord 메시지 길이 제한 (2000자) 준수
- 에이전트 간 무한 루프 방지 (최대 3회 반복)
- 각 단계 완료 후 사용자 확인 가능
